## 一、功能增强

### 1. 历史消息导出入口

- 优先级: `P0`
- 原因: Main/Preload 已经支持 `txt/csv` 导出，但当前界面没有对应入口，已有能力尚未转化为用户功能。
- 实现方案:
  - 在顶栏或历史菜单增加“导出消息”按钮。
  - 支持导出为 `txt` 和 `csv` 两种格式。
  - Renderer 调用现有 `window.electronAPI.exportMessages()`，再配合系统保存对话框落盘。
  - 导出成功后给出 Toast 提示，失败时复用已有错误提示体系。

### 2. 单条消息删除

- 优先级: `P0`
- 原因: 当前只支持“清空全部历史”，误删成本高，且单条删除的底层 IPC 和 Storage 能力已存在。
- 实现方案:
  - 在 `MessageBubble` 增加删除按钮和确认交互。
  - 调用已有 `deleteMessage` IPC，成功后从 `messageStore` 中移除对应消息。
  - 对流式消息状态和普通消息统一处理，避免删除时留下悬空状态。

### 3. 模型下载后的自动续用

- 优先级: `P0`
- 原因: 当前首次使用语音/翻译时遇到缺模型，用户需要重新触发一次操作，流程有割裂感。
- 实现方案:
  - 记录用户触发动作上下文，例如“当前正在开始录音”或“当前消息待翻译”。
  - 模型下载或安装完成后，自动恢复原始动作。
  - Whisper 场景下，下载完成后直接继续启动录音；翻译场景下，安装完成后自动重试当前翻译请求。

### 4. 模型占用空间展示与清理策略

- 优先级: `P1`
- 原因: 当前模型管理器只展示安装状态，不展示磁盘占用，也无法帮助用户做空间治理。
- 实现方案:
  - 在主进程增加模型目录大小统计接口。
  - 在 `ModelManagerDialog` 中显示每个语言对模型大小、总占用空间。
  - 增加“删除最近最少使用模型”或“清理未使用模型”的批量清理入口。

### 5. UI 国际化

- 优先级: `P1`
- 原因: 应用支持多语种识别和翻译，但界面文案目前基本固定为中文，不利于非中文用户使用。
- 实现方案:
  - 提取 Renderer 中硬编码文案，建立 `i18n` 文案字典。
  - 先覆盖 `zh-CN` / `en` 两套界面文案。
  - 设置层增加界面语言配置，并把语言设置持久化到本地。

### 6. 演示截图与文档素材整理

- 优先级: `P2`
- 原因: README 当前没有仓库内可用截图，影响项目展示和传播。
- 实现方案:
  - 新建 `docs/images/` 目录。
  - 补充主界面、模型管理器、流式识别三个核心截图。
  - README 中改为使用仓库相对路径资源。

## 二、性能优化

### 7. 翻译服务预热

- 优先级: `P0`
- 原因: 性能分析文档已明确“首次翻译延迟”是当前主要体验损耗之一，且 Whisper 已实现预热，翻译服务尚未对齐。
- 实现方案:
  - 在应用 `ready` 后或首次打开翻译开关时，后台启动 `TranslateService`。
  - 增加空闲优先级调度，避免与启动首屏争抢资源。
  - 预热失败只记录日志，不打断主流程。

### 8. 长列表虚拟滚动

- 优先级: `P1`
- 原因: 当前消息列表按 DOM 全量渲染，历史记录规模上来后会增加内存和滚动压力。
- 实现方案:
  - 在 `ChatView` 中引入 `react-window` 或同类方案。
  - 为普通消息和流式消息设计统一高度策略，必要时使用动态高度缓存。
  - 保留现有“向上分页加载”能力，处理好 prepend 后的滚动位置恢复。

### 9. 空闲时自动卸载常驻推理进程

- 优先级: `P1`
- 原因: `whisper-server` 预热后可降低首响延迟，但也带来持续内存占用；翻译服务同样存在模型驻留成本。
- 实现方案:
  - 为 Whisper 和翻译服务增加空闲计时器，例如 10-15 分钟无请求则自动关闭。
  - 再次使用时按需拉起，并与“预热策略”配套设计开关。
  - 设置页增加“优先响应 / 优先省内存”模式供用户选择。

### 10. Whisper 量化模型支持

- 优先级: `P1`
- 原因: `q5_0` 等量化模型能显著减少内存与推理时间，尤其适合中低配设备。
- 实现方案:
  - 模型管理器增加 Whisper 模型规格选择，如 `base` / `base-q5_0`。
  - 下载层按模型规格区分 URL、文件名和校验值。
  - UI 中展示“速度优先 / 精度优先”的推荐说明。

### 11. 翻译模型缓存与 LRU 策略

- 优先级: `P1`
- 原因: 频繁切换目标语言时，翻译服务的模型载入会造成反复等待。
- 实现方案:
  - 在翻译服务中记录最近使用的语言对。
  - 评估单进程多模型缓存或按语言对建立 LRU 缓存。
  - 限制缓存上限，避免内存无限增长。

### 12. Windows NVIDIA 路径下的 GPU 翻译支持

- 优先级: `P2`
- 原因: 当前 CTranslate2 主要还是 CPU 路径，Windows NVIDIA 用户有明显加速空间。
- 实现方案:
  - 评估 CUDA 版 `ctranslate2` 的打包与分发策略。
  - 区分 CPU / CUDA 运行时，启动时自动检测可用设备。
  - 文档层提供显卡驱动与 CUDA 安装指引。

## 三、工程质量与可维护性

### 13. 补齐端到端测试

- 优先级: `P0`
- 原因: 当前已有 Vitest 和 Playwright 依赖，但缺少覆盖关键路径的 E2E 用例，回归风险高。
- 实现方案:
  - 先覆盖三个核心路径：文字发送与翻译、模型管理安装流程、语音按钮权限与异常提示。
  - 为外部依赖建立 mock 或 stub，避免测试中真的拉起 Whisper / Python 推理。
  - 在 CI 中拆分为轻量单元测试和关键 E2E 冒烟测试。

### 14. 建立 CI 构建与测试流水线

- 优先级: `P0`
- 原因: 当前打包、测试和文档更新依赖本地手工执行，交付稳定性不足。
- 实现方案:
  - 新增 GitHub Actions：`pnpm install`、`pnpm test`、`pnpm build`。
  - 对 macOS 增加独立的打包校验工作流。
  - 对 README / Markdown 做基础 lint，降低文档漂移。

### 15. 性能报告可视化面板

- 优先级: `P1`
- 原因: `PerformanceService` 已能输出数据，但当前只能通过 DevTools Console 调用，无法直接用于日常验证。
- 实现方案:
  - 在设置面板或开发者模式中加入性能页。
  - 展示启动耗时、最近几次翻译耗时、Whisper 耗时、当前 RSS 等关键指标。
  - 增加手动触发 `take-snapshot` 的按钮，辅助问题定位。

### 16. 清理未接通或历史残留能力

- 优先级: `P1`
- 原因: 代码中仍存在部分未真正接通到主流程的能力或旧路径，后续维护成本会逐渐抬高。
- 实现方案:
  - 审视 `PythonSetupGuide`、历史 IPC 能力和预留接口，确认哪些要接入、哪些应删除。
  - 对 Renderer / Preload / Main 三层做一次接口收敛，避免出现“类型里有、实现里无”或“实现里有、UI 不用”的漂移。
  - 清理后补一轮类型测试和文档同步。

### 17. 错误恢复与诊断信息增强

- 优先级: `P1`
- 原因: 当前已有自动重启和错误提示，但对用户来说还不够可诊断，出现问题时定位成本高。
- 实现方案:
  - 为 Whisper / 翻译服务失败增加错误码和分级提示。
  - 在日志中记录服务启动、退出、重启次数与最近错误摘要。
  - 在 UI 中增加“查看诊断信息 / 复制错误详情”入口。

## 四、发布与交付

### 18. 自动更新链路

- 优先级: `P1`
- 原因: 当前已经有 `latest-mac.yml` 等打包产物，但文档和应用侧还没有完整的自动更新体验。
- 实现方案:
  - 接入 Electron Auto Updater。
  - 区分稳定版与测试版更新源。
  - 增加更新检查、下载进度、重启安装提示。

### 19. macOS / Windows 发布检查清单

- 优先级: `P1`
- 原因: 当前打包链路涉及签名、语音权限、内置二进制等多个平台差异点，缺少统一的发布 checklist。
- 实现方案:
  - 新建发布文档，覆盖证书、签名、权限描述、路径验证和 smoke test 项。
  - 将 `electron:build:voice` 相关检查前置到发布流程。
  - 对 release 产物建立最小可运行验证脚本。

### 20. 安装后首启引导

- 优先级: `P2`
- 原因: 应用能力较多，首次用户需要理解麦克风权限、模型下载和翻译开关的关系，否则容易卡在“为什么不能用”。
- 实现方案:
  - 增加首启引导页或分步提示。
  - 逐步解释麦克风权限、Whisper 模型下载、翻译模型安装与目标语言选择。
  - 引导结束后持久化状态，后续不再重复展示。
